# Parsing

## 두 가지 과정
> * Lexical Analysis : tokenizer을 이용해 원시 코드를 token으로 나눔.
> * Syntactic Analysis : token을 원시 코드의 문법과 관계를 표현할 수 있게 조합함. **intermediate representation** 또는 **Abstract Syntax Tree**라고 불림.

Abstarct Syntax Tree : deeply nested object. 코드에 대해 많은 정보를 제공하고 처리하기 쉽게 코드를 가공한 형태

***
다음 코드에 대해 위의 두 과정을 거치면

```
 * For the following syntax:
 *
 *   (add 2 (subtract 4 2))
```

## Lexical Analysis
```
 *   [
 *     { type: 'paren',  value: '('        },
 *     { type: 'name',   value: 'add'      },
 *     { type: 'number', value: '2'        },
 *     { type: 'paren',  value: '('        },
 *     { type: 'name',   value: 'subtract' },
 *     { type: 'number', value: '4'        },
 *     { type: 'number', value: '2'        },
 *     { type: 'paren',  value: ')'        },
 *     { type: 'paren',  value: ')'        },
 *   ]
```

## Syntactic Analysis
```
 *   {
 *     type: 'Program',
 *     body: [{
 *       type: 'CallExpression',
 *       name: 'add',
 *       params: [{
 *         type: 'NumberLiteral',
 *         value: '2',
 *       }, {
 *         type: 'CallExpression',
 *         name: 'subtract',
 *         params: [{
 *           type: 'NumberLiteral',
 *           value: '4',
 *         }, {
 *           type: 'NumberLiteral',
 *           value: '2',
 *         }]
 *       }]
 *     }]
 *   }
```
